### 第四步：配置 MySQL 与 Prisma 模型

接下来我们配置一下后端的数据库连接和表结构。

1.  **配置 `.env` 环境变量**：
    用编辑器打开 `server/.env`，将其中的 `DATABASE_URL` 修改为你本地 MySQL 的信息（请先确保本地 MySQL 服务已启动，且存在名为 `coboard_db` 的空数据库）：

    ```env
    DATABASE_URL="mysql://root:你的密码@localhost:3306/coboard_db"
    PORT=3000
    JWT_SECRET="coboard_secret_key_123"
    ```

2.  **编写数据库模型 (`server/prisma/schema.prisma`)**：
    打开 `schema.prisma`，将内容替换为以下符合 **CoBoard** 逻辑的模型设计：

    ```prisma
    datasource db {
      provider = "mysql"
      url      = env("DATABASE_URL")
    }
    
    generator client {
      provider = "prisma-client-js"
    }
    
    // 用户表
    model User {
      id        Int      @id @default(autoincrement())
      email     String   @unique
      password  String
      name      String
      createdAt DateTime @default(now())
      updatedAt DateTime @updatedAt
    
      boards    Board[]  @relation("CreatedBoards")
    }
    
    // 看板表
    model Board {
      id          Int      @id @default(autoincrement())
      title       String
      description String?
      userId      Int
      createdAt   DateTime @default(now())
      updatedAt   DateTime @updatedAt
    
      owner       User     @relation("CreatedBoards", fields: [userId], references: [id], onDelete: Cascade)
      lists       List[]
    }
    
    // 列表表（待处理、进行中、已完成等）
    model List {
      id        Int      @id @default(autoincrement())
      title     String
      position  Int
      boardId   Int
      createdAt DateTime @default(now())
      updatedAt DateTime @updatedAt
    
      board     Board    @relation(fields: [boardId], references: [id], onDelete: Cascade)
      cards     Card[]
    }
    
    // 卡片表（具体任务）
    model Card {
      id          Int      @id @default(autoincrement())
      title       String
      description String?  @db.Text
      position    Int
      listId      Int
      createdAt   DateTime @default(now())
      updatedAt   DateTime @updatedAt
    
      list        List     @relation(fields: [listId], references: [id], onDelete: Cascade)
    }
    ```

3.  **运行数据库迁移**：
    在 `server` 目录下运行迁移命令，将上述结构同步到本地 MySQL：
    ```bash
    cd server
    npx prisma migrate dev --name init
    cd ..
    ```

### 第五步：编写后端入口与测试接口

1.  在 `server` 目录下创建 `src` 文件夹，并在其内新建 `index.js`：

    ```bash
    mkdir -p server/src
    touch server/src/index.js
    ```

2.  打开 `server/src/index.js`，写入以下基础代码：

    ```javascript
    const express = require("express");
    const cors = require("cors");
    const { PrismaClient } = require("@prisma/client");
    require("dotenv").config();

    const app = express();
    const prisma = new PrismaClient();
    const PORT = process.env.PORT || 3000;

    app.use(cors());
    app.use(express.json());

    // 测试路由：获取所有看板
    app.get("/api/boards", async (req, res) => {
      try {
        const boards = await prisma.board.findMany();
        res.json({ success: true, data: boards });
      } catch (error) {
        res.status(500).json({ success: false, message: error.message });
      }
    });

    app.listen(PORT, () => {
      console.log(`Server is running on port ${PORT}`);
    });
    ```

3.  在 `server/package.json` 的 `scripts` 中添加启动命令：
    ```json
    "scripts": {
      "dev": "nodemon src/index.js"
    }
    ```

### 第六步：利用 pnpm 统一启动前后端

得益于 pnpm workspace，你不再需要像使用 npm 那样单独安装 `concurrently`。我们可以直接利用 pnpm 提供的多包并行运行能力。

1.  **检查 `client/package.json` 中的启动脚本**：
    确认其包含：

    ```json
    "scripts": {
      "dev": "vite"
    }
    ```

2.  **配置根目录的 `package.json`**：
    打开**根目录下的 `package.json`**，将其修改为：
    ```json
    {
      "name": "coboard-monorepo",
      "version": "1.0.0",
      "description": "CoBoard Workspace",
      "private": true,
      "scripts": {
        "dev": "pnpm -r --parallel dev"
      }
    }
    ```
    _说明：`pnpm -r --parallel dev` 会递归（-r）寻找所有子包（client 和 server）并并行（--parallel）执行它们各自 package.json 中的 `dev` 脚本。_

---

### 启动项目！

现在，在你的**项目根目录**下，只需执行一行命令：

```bash
pnpm dev
```

终端将会同时输出前端（Vite）和后端（Nodemon）的启动日志。

- 前端通常会运行在 `http://localhost:5173`。
- 后端会运行在 `http://localhost:3000`。

现在你的 **CoBoard** 全栈项目基础环境已经完全搭建完毕，你可以开始按模块逐步实现看板的协同逻辑了。
