# CoBoard —— 协同任务看板系统

**CoBoard**（意为 Collaborative Board）是一款仿 Trello 的**多人协同任务看板系统**。本项目采用前后端分离的单体仓库（Monorepo）架构，旨在通过完整的全栈开发实践，深度磨练 Vue 3 前端工程化、Express 接口设计以及关系型数据库的建模与治理能力。

---

## 🛠 核心技术栈

项目基于当前主流且实用的全栈开发技术进行构建：

- **项目架构**：Monorepo（单体仓库，使用 `pnpm workspace` 统一管理）
- **前端（client）**：
  - **框架**：Vue 3（采用 Composition API）
  - **构建工具**：Vite
  - **状态管理**：Pinia（用于看板复杂状态树的管理）
  - **路由管理**：Vue Router 4（实现路由守卫与鉴权控制）
  - **UI 库/样式**：Tailwind CSS
  - **核心库**：`vuedraggable`（实现无缝拖拽）
- **后端（server）**：
  - **框架**：Express.js（轻量级 Node.js 框架）
  - **ORM（对象关系映射）**：Prisma（配合 TypeScript/JavaScript 强类型类型提示）
  - **数据库**：MySQL 9.x
  - **安全/认证**：JSON Web Token (JWT)、`bcrypt`（密码哈希）
  - **实时通信**：Socket.io（用于多人实时协同数据同步，属进阶模块）

## 📂 项目结构设计

项目采用规范的单体仓库架构，使前后端代码在同一个 Git 仓中，避免了管理两个仓库的繁琐：

```text
CoBoard/
├── client/              # 前端 Vue 3 应用程序
├── server/              # 后端 Express 应用程序
├── doc/                 # 过程开发文档/设计底稿
├── pnpm-workspace.yaml  # pnpm 空间定义
├── package.json         # 根级 package.json（通过一行指令并行启动前后端）
└── .gitignore           # 全局 Git 忽略规则文件
```

## 🎯 业务功能清单

系统主要围绕 **看板（Board）-> 列表（List）-> 卡片（Card）** 这一经典的三层数据模型展开，具体功能如下：

### 1. 用户认证与账号体系

- **注册与登录**：支持邮箱/密码登录，前端进行即时表单校验。
- **安全加固**：后端使用 `bcrypt` 算法对用户密码进行强哈希存储，确保数据库被拖库时密码也不易泄露。
- **会话管理**：基于 JWT 机制，登录成功后下发 Token。前端将 Token 妥善保管并在每次请求的 `Authorization` 请求头中携带，实现无状态的鉴权机制。

### 2. 看板与列表层级管理

- **个人工作区**：用户登录后进入个人主页，可创建新看板，并查看自己创建或受邀参与的所有看板。
- **列表 CRUD**：进入某一看板后，用户可水平创建多个列表（例如：待处理、进行中、已完成），支持列表重命名与删除。
- **位置排序**：支持列表之间的水平拖拽，以调整不同工作阶段的展示顺序。

### 3. 卡片与任务交互（核心）

- **快速创建**：支持在指定列表底部快速输入标题创建卡片（Task）。
- **卡片拖拽**：
  - _同列拖拽_：在单个列表内上下拖拽，手动调整任务的优先级（高、中、低）。
  - _跨列拖拽_：支持卡片在不同列表之间自由移动，模拟任务状态流转（例如将卡片从“进行中”拖到“已完成”）。
- **详情面板（Modal）**：点击卡片可唤出大图，进行深度编辑：
  - **描述**：支持 Markdown 格式的富文本输入。
  - **标签（Labels）**：通过可定制颜色的分类标签标记卡片属性。
  - **截止日期**：设置任务截止时间，提供接近期、已过期等状态的高亮提醒。
  - **检查清单（Checklist）**：支持在卡片内拆分二级子任务，动态显示进度条。
  - **评论区（Comments）**：团队成员可以在卡片底部留言交流。

### 4. 多人协作与实时同步（进阶拓展）

- **成员邀请**：看板拥有者（Owner）可以通过邮箱搜索其他用户，并将其添加为该看板的协作成员。
- **实时更新（Socket.io）**：
  - 多人在同一个看板中工作时，任何一人拖动卡片、修改标题或添加评论，其他人的视图将即时发生相应变化，而无需频繁手动刷新。

## 💡 全栈实践带来的核心收获

通过开发此项目，你可以较为扎实地掌握以下全栈工程化技能：

1.  **复杂数据状态管理**：如何用 Pinia 优雅管理看板中 `看板-列表-卡片` 的嵌套树状数据，以及本地状态修改与后端 API 同步的性能平衡（如：乐观更新 Optimistic UI）。
2.  **优雅的数据库建模**：学会使用 ORM（Prisma）设计清晰的关系型数据库 Schema，并处理外键级联删除（Cascade）和数据关联查询。
3.  **前后端安全规范**：掌握基于 JWT 的路由守卫、跨域资源共享（CORS）的安全配置，以及数据库敏感字段（密码）的安全防护。
4.  **现代 Monorepo 工程体验**：体验 `pnpm workspace` 带来的极速依赖安装和多服务并行的流畅开发过程。
